量子 集群/服务器控制台 详细说明文档

一、软件定位

这是一个 Windows 原生桌面程序，用 SSH 管理 Linux 服务器或小型集群。
它适合没有 Slurm/PBS 的轻量环境，也可以作为已有集群的简易可视化入口。

程序不内置任何服务器地址、账号或密码。第一次打开时需要自己填写连接信息。

二、连接方式

1. 直接连接

适用场景：
本机电脑可以直接访问目标服务器的 SSH 端口。

连接结构：
本机电脑 -> 目标服务器

需要填写：
直连主机：目标服务器 IP 或域名。
端口：SSH 端口，常见为 22。
账号：Linux 用户名。
密码：Linux 密码。

2. 跳板机方式

适用场景：
目标计算节点在内网，本机不能直接访问，需要先登录一台跳板机。

连接结构：
本机电脑 -> 跳板机 -> 内网计算节点

需要填写：
跳板机：跳板机 IP 或域名。
跳板机端口：跳板机 SSH 端口。
内网节点：从跳板机能够访问到的计算节点地址。
内网节点端口：内网节点 SSH 端口，常见为 22。
账号：Linux 用户名。
密码：Linux 密码。

当前版本默认使用同一组账号密码登录跳板机和内网节点。

三、保存集群

登录页可以保存多个集群配置。

每个配置包含：
集群名称
连接方式
跳板机地址和端口
目标节点地址和端口
账号
是否尝试本机 SSH 密钥
可选的加密密码

如果不勾选“记住账号和密码”，程序只保存地址和账号，不保存密码。
如果勾选“记住账号和密码”，密码会使用 Windows 当前用户的数据保护能力保存到本机。

保存的数据默认位于程序目录下的 data 文件夹。

四、总览页

总览页会通过 SSH 执行只读命令，读取：
CPU 使用率
内存使用量
磁盘使用量
平台作业数量和外部任务数量
节点名称
CPU 核心数
运行时间
是否检测到 Slurm/PBS 命令

这些信息只是显示，不会修改远程系统。

五、作业页

作业页提供一个轻量作业系统，不依赖 Slurm/PBS。
作业表会合并显示“平台”和“外部”两类任务。

提交作业时需要填写：
名称：给任务起一个名字。
目录：命令在哪个远程目录中运行。
命令：要执行的 shell 命令。

示例：
bash run.sh
python train.py > out.txt 2> err.txt
./main > out.txt

平台提交的作业会在远程用户目录下创建：
~/.cluster_panel/

主要内容：
jobs/ 保存作业状态、pid、stdout、stderr。
bin/run_job.sh 负责后台启动和记录状态。

平台作业：
通过本软件提交。
支持状态、日志和停止按钮。

外部任务：
由同一账号从其他 SSH 终端、nohup 或脚本手动启动。
程序会扫描当前账号的计算进程，显示 PID、CPU 核数、运行时间、目录和命令。
外部任务不强制寻找 stdout/stderr，因为输出通常已经写入用户指定的文件。
如果需要查看输出，可以在“文件”页进入对应目录查看。

停止作业：
点击“停止”会真实向远程进程发送终止信号。
程序会优先尝试 TERM，必要时再尝试 KILL。

注意：
只有平台作业有完整的平台状态和日志。
外部任务能显示运行状态和资源占用，但已经结束的外部任务通常无法追溯。

六、文件页

文件页通过 SFTP 操作远程文件。

支持：
打开目录
返回上级
新建目录
上传文件
下载文件
文本预览
改名
删除文件或空目录

默认隐藏点文件，例如 .cache、.config、.ssh。
勾选“显示隐藏文件”后可以显示。

删除操作会真实删除远程文件，请谨慎使用。

七、命令行页

命令行页通过当前 SSH 会话执行普通 shell 命令。

可用示例：
pwd
ls -lh
cd project
tail -n 50 out.txt
cat config.json

命令执行后会显示 stdout、stderr 和退出码。
cd 命令会保留当前目录，下次命令会从新的目录继续执行。
clear 会清空本地显示区域。

限制：
当前命令行不是完整 PTY 终端。
不建议直接运行 vim、top、htop、less 这类全屏交互程序。
长时间计算建议使用“作业”页，或者自行使用 nohup：
nohup python run.py > out.txt 2>&1 &

八、安全说明

程序不会内置默认服务器。
程序不会把密码写入源码。
程序不会把密码写入日志。
程序不会要求 sudo。
程序不会安装远程系统服务。

保存密码只对当前 Windows 用户可读。
把程序文件夹复制给别人时，建议不要复制 data 文件夹。

九、源码结构

native_app.py
桌面界面入口，使用 PySide6。

backend/app/ssh_client.py
SSH 连接层，支持直接连接和跳板机 direct-tcpip 通道。

backend/app/remote.py
远程指标、作业、文件、命令行等逻辑。

backend/app/sessions.py
登录凭据和会话数据结构。

backend/app/config.py
应用名称、默认端口、资源路径、数据路径。

assets/
软件图标和背景资源。

tests/
单元测试和 UI 逻辑测试。

十、开发运行

python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\run_dev.ps1

十一、打包

.\build_exe.ps1

打包结果：
dist\量子集群服务器控制台\量子集群服务器控制台.exe
